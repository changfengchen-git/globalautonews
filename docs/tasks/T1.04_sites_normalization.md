# T1.04 — 站点列表规范化 (sources.yaml)

## 任务目标
将 `sites/sites0415` 中的原始站点列表转换为结构化 YAML 配置文件。

## 前置依赖
无（可与其他任务并行）

## 工作目录
`/Users/apple/Desktop/globalautonews/`

## 输入文件
`sites/sites0415` — 原始站点列表，格式不统一（有完整URL和纯域名混合）

## 输出文件
`config/sources.yaml`

## 转换规则

### 1. 格式要求
每个站点一个条目，字段如下：

```yaml
sources:
  - url: "https://www.example.com/auto/news/"   # 必填，新闻列表页完整URL
    name: "Example Auto"                         # 必填，站点显示名称
    domain: "example.com"                        # 必填，域名（不含www）
    country: "US"                                # 必填，ISO 3166-1 alpha-2
    language: "en"                               # 必填，ISO 639-1（可带地区：zh-HK）
    region: "North America"                      # 可选，区域分组
    priority: "high"                             # high | medium | low
    tier: 1                                      # 1=通用提取, 2=需适配器, 3=需定制
```

### 4. 去重规则
- **全球站点去重**：motor1.com, motorsport.com, caranddriver.com, topgear.com 等在多个区域重复出现的站点，只保留一条记录，country 设为站点主要市场
- **本地化版本独立保留**：tr.motor1.com, motor1.uol.com.br 等各国版本独立保留
- **排除项**：
  - Facebook 页面（carnewsphilippines）→ 不纳入，标注为 Tier 3 待开发
  - justanswer.com → 问答站非新闻站，排除
  - freep.com → 地方报纸，汽车新闻覆盖有限，priority 设为 low

### 5. 优先级初始化规则
- **high**：该国/语种最知名的 2-3 个汽车媒体
- **medium**：一般新闻站的汽车频道
- **low**：地方性或低更新频率的站点
- 默认 `medium`，如不确定
