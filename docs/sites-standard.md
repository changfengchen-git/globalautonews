## 站点文件格式规范

### 文件位置

```
sites/sites0415    # 主站点文件
```

### 格式规则

```
# 每行一个站点，支持以下格式：

# 格式1: 完整 URL（推荐）
https://www.example.com/news
http://example.com/auto

# 格式2: 域名
example.com
news.example.com

# 注释以 # 开头
# 这是注释

# 空行会被忽略
```

### 推荐的文件结构

```
# ==========================================
# 地区名称（如：印度 India）
# ==========================================

# 子分类（如：英语媒体）
https://www.autocarindia.com/
cardekho.com

# 子分类（如：印地语媒体）
https://www.carwale.com/hi/news/
https://hindi.drivespark.com/

# ==========================================
# 地区名称（如：日本 Japan）
# ==========================================
carview.co.jp
bestcarweb.jp
```

## 添加新站点流程

### 方式 1: 编辑 sites 文件（推荐批量添加）

```bash
# 1. 编辑文件
vim /Users/apple/Desktop/globalautonews/sites/sites0415

# 2. 添加站点（每行一个）
https://www.newsite.com/auto
newsite2.com

# 3. 运行导入脚本
cd /Users/apple/Desktop/globalautonews
DATABASE_URL="postgresql+asyncpg://gan:password@localhost:5432/globalautonews" \
  python3 scripts/import_sites.py sites/sites0415
```

### 方式 2: 通过 API（单个添加）

```bash
curl -X POST "http://localhost:8000/api/sources" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "New Auto Site",
    "domain": "newsite.com",
    "url": "https://newsite.com/auto",
    "country": "US",
    "language": "en",
    "priority": "medium",
    "tier": 1
  }'
```

### 方式 3: 通过前端

访问 http://localhost:80/candidates → 候选审批页面

## 站点属性说明

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 是 | 站点名称 |
| `domain` | string | 是 | 域名（唯一标识） |
| `url` | string | 是 | 首页/列表页 URL |
| `country` | string | 是 | 国家代码 |
| `language` | string | 是 | 语言代码 |
| `tier` | int | 否 | 1=通用, 2=适配器, 3=定制 |
| `rendering` | string | 否 | static/dynamic |
| `priority` | string | 否 | high/medium/low |

## 站点管理

### 查看站点列表

访问: http://localhost:80/sources

### API 端点

```bash
# 获取站点列表
curl "http://localhost:8000/api/sources"

# 获取单个站点
curl "http://localhost:8000/api/sources/1"

# 更新站点
curl -X PATCH "http://localhost:8000/api/sources/1" \
  -H "Content-Type: application/json" \
  -d '{"status": "paused"}'

# 重试站点
curl -X POST "http://localhost:8000/api/sources/1/retry"

# 升级到 Tier 2
curl -X POST "http://localhost:8000/api/sources/1/upgrade-tier"
```
