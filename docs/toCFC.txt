一、你需要准备的环境
========================================

1. 一台 VPS（4GB 内存推荐，2GB 也能跑精简模式）
2. VPS 上安装 Docker 和 Docker Compose
3. 本地开发环境：Python 3.11+、Node.js 18+（如果要本地调试的话）
4. 一个 OpenAI API Key（用于自我生长模块的 LLM 分析，GPT-4o-mini 即可）
5. 一个 DeepL API Key（可选，用于翻译功能）

========================================

【第一步】每次新开一个 AI 编程会话时，先让它读这两个文件：
  - docs/AI_CODING_GUIDE.md
  - docs/tasks/README.md

  参考 Prompt：
  "请先阅读这两个文件了解项目全局，然后我会给你具体的任务文档。"

【第二步】一次给一个任务。参考 Prompt：

  "请阅读以下任务文档并执行：

  [粘贴任务文档全文]

  项目根目录：/Users/apple/Desktop/globalautonews/
  请严格按照文档要求创建文件，完成后运行验收标准中的测试。

  注意事项：
  - Python 3.11+，使用 type hints
  - 使用 async/await 异步编程

【第三步】每个任务完成后，检查验收标准是否全部通过，再进入下一个任务。

五、关键注意事项
========================================

1. 【内存】你的 VPS 只有 4GB 内存，每个容器都有内存上限：
   - PostgreSQL: 512MB
   - 爬虫: 1536MB（含 Playwright 和嵌入模型）
   - API: 256MB
   - Nginx: 64MB
   如果是 2GB VPS，用 docker-compose.lite.yml 覆盖配置。

2. 【图片不要存文件】系统只存图片的 URL 链接，不下载图片文件。
   50GB 硬盘存不了多少图片的（每天 200MB，8个月就满了）。
   前端直接引用原站图片的 URL 展示。

3. 【嵌入模型】用 EmbeddingGemma (308M)，约 350MB 内存。
   4GB VPS 可以本地跑，2GB VPS 改用 API 调用（月增约 $3）。
   默认用 256 维向量（MRL 技术），精度不够可以升到 512 维。

4. 【API 成本】稳态运行每月约 $15-25：
   - 四级去重的前三级不花钱（本地计算）
   - 第四级向量去重 4GB 模式下也不花钱（本地模型）
   - LLM 只用在"自我生长"分析新站点（每天最多20个，每个$0.005）
   - 翻译是按需的，不翻译就不花钱

5. 【站点列表】sites0415 里有些站点只有域名没有完整 URL，
   T1.04 任务里有详细的补全规则和对照表。
   初始大概能整理出 160-180 个有效站点。

6. 【自动清理】站点不是只增不减的：
   - 连续出错 5 次 → 降频观察
   - 连续出错 15 次 → 暂停
   - 90 天无新文章 → 自动归档
   - 每周日凌晨会对暂停站点做探活检测

7. 【Phase 1 完成就能用了】不需要等全部 5 个 Phase 做完。
   Phase 1 做完：能抓取、能去重、能通过 API 查询
   Phase 2 做完：有前端界面、有智能去重
   Phase 3 做完：系统会自己发现新站点
   Phase 4-5：锦上添花

8. 【有些文件会被多个任务修改】主要是：
   - crawler/main.py：T1.01 创建骨架 → T1.10 写完整实现
   - crawler/pipeline/dedup.py：T1.09 写L1+L2 → T2.03 加L3 → T2.04 加L4
   - crawler/engine/fetcher.py：T1.06 写HTTP → T2.01 加Playwright
   所以后续任务需要先看看文件当前什么样，再改。


六、如果遇到问题
========================================

1. 某个任务做不下去 → 记录错误信息，下次会话带上继续
2. AI 工具改错了之前的文件 → 用 git 回滚，重新执行当前任务
3. 数据库连不上 → 先确保 docker compose up -d postgres 跑起来了
4. 内存不够 → 用 docker-compose.lite.yml 精简模式
5. 某个站点抓不到 → 正常，先跳过，后续 Playwright（T2.01）或 YAML 适配器（T4.03）解决
6. 方案需要调整 → 随时可以改 settings.yaml 里的参数，大多数是可配置的
